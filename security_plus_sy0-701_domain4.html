4.1 Common Security Techniques for Computing Resources

Secure Baselines

  • Establish — define a known-good secure configuration.
  • Deploy — push the baseline to systems (imaging, GPO, MDM).
  • Maintain — monitor for drift and re-apply.

Hardening Targets

Mobile, workstations, switches, routers, cloud infrastructure, servers, ICS/SCADA, embedded systems, RTOS, IoT.

Mobile Solutions

  • MDM (Mobile Device Management) — central control: policy, remote wipe, encryption, app control.
  • Deployment models:
    • BYOD — Bring Your Own Device (employee-owned).
    • COPE — Corporate-Owned, Personally Enabled.
    • CYOD — Choose Your Own Device (from an approved list).
  • Connection methods — cellular, Wi-Fi, Bluetooth.

Wireless Security

  • WPA3 — current standard; uses SAE (Simultaneous Authentication of Equals) to replace the WPA2 4-way handshake.
  • AAA / RADIUS — enterprise wireless auth (802.1X).
  • Cryptographic & authentication protocols — EAP-TLS, PEAP, etc.

Application Security

  • Input validation — reject malformed/malicious input (defends SQLi, XSS).
  • Secure cookies — Secure + HttpOnly flags.
  • Static code analysis (SAST) — scan source code without running it.
  • Code signing — verify integrity/origin of software.
  • Sandboxing — run untrusted code in an isolated environment.
  • Monitoring — continuous observation of resource behavior.

4.2 Asset Management (Hardware, Software, Data)

  • Acquisition / procurement — vetting and securely purchasing assets.
  • Assignment / accounting — ownership (who is responsible) and classification (sensitivity).
  • Monitoring / asset tracking — inventory (what we have) and enumeration (discovering devices on the network).
  • Disposal / decommissioning:
    • Sanitization — securely erase data (crypto-erase, overwrite).
    • Destruction — physically destroy media (shred, degauss, incinerate).
    • Certification — documented proof of destruction/sanitization.
    • Data retention — keep data only as long as policy/law requires.

4.3 Vulnerability Management

Identification Methods

  • Vulnerability scan — automated discovery of weaknesses (credentialed vs. non-credentialed).
  • Application security — SAST (static), DAST (dynamic, runs the app), package/dependency monitoring.
  • Threat feeds — OSINT, proprietary/third-party feeds, info-sharing orgs (ISACs), dark web monitoring.
  • Penetration testing — authorized simulated attack.
  • Responsible disclosure / bug bounty — researchers report flaws (often for reward).
  • System / process audit.

Analysis

  • Confirmation: true positive, false positive (flagged but not real), false negative (missed real vuln), true negative.
  • Prioritize — rank by risk.
  • CVSS — Common Vulnerability Scoring System (0–10 severity).
  • CVE — Common Vulnerabilities and Exposures (unique IDs).
  • Exposure factor, environmental variables, industry/organizational impact, risk tolerance.

Response & Remediation

  • Patching, insurance, segmentation, compensating controls, exceptions / exemptions (formally accept the risk).

Validation & Reporting

  • Validation — rescanning, audit, verification confirm the fix worked.
  • Reporting — communicate findings/metrics to stakeholders.

4.4 Security Alerting & Monitoring

Monitoring Resources

Systems, applications, and infrastructure.

Activities

  • Log aggregation — collect logs centrally.
  • Alerting — notify on suspicious events.
  • Scanning, reporting, archiving.
  • Alert response & remediation/validation:
    • Quarantine — isolate the affected system.
    • Alert tuning — adjust rules to cut false positives/noise.

4.5 Modify Enterprise Capabilities to Enhance Security

  • Firewall — rules, access lists (ACLs), ports/protocols, screened subnets (DMZ).
  • IDS / IPS — trends and signatures.
  • Web filter — agent-based, centralized proxy, URL scanning, content categorization, block rules, reputation.
  • OS security — Group Policy (Windows), SELinux (Linux mandatory access control).
  • Secure protocols — protocol selection (use SSH not Telnet, HTTPS not HTTP), port selection, transport method.
  • DNS filtering — block malicious domains at resolution.
  • Email security:
    • SPF — authorizes which servers may send for a domain.
    • DKIM — cryptographically signs messages.
    • DMARC — policy + reporting built on SPF/DKIM (what to do on failure).
    • Email gateway — filters spam/malware/phishing.
  • FIM (File Integrity Monitoring) — alerts on unauthorized file changes.
  • DLP — prevent data exfiltration.
  • NAC (Network Access Control) — posture-check and authorize devices before granting access.
  • EDR / XDR — Endpoint (and eXtended) Detection & Response.
  • UBA (User Behavior Analytics) — baseline behavior, alert on anomalies.

4.6 Identity and Access Management (IAM)

Account Lifecycle

  • Provisioning / de-provisioning — create on hire, disable/remove on termination.
  • Permission assignments, identity proofing (verify who someone is).

Federation & SSO

  • Federation — trust identities across organizations/domains.
  • SSO — one login for many services.
  • LDAP — directory access protocol.
  • OAuth — authorization (delegated access via tokens).
  • SAML — XML-based authentication/SSO for web apps.
  • Attestation — periodic verification that access rights are still appropriate.

Access Control Models

ModelBasis
MACMandatory — labels/clearances set by system (military).
DACDiscretionary — data owner sets permissions.
RBACRole-Based — permissions tied to job role.
Rule-basedAccess via predefined rules (e.g., firewall ACLs).
ABACAttribute-Based — evaluates attributes (user, resource, environment).
Time-of-dayRestrict access to allowed hours.
Least privilegeMinimum access necessary.

Multifactor Authentication (MFA)

  • Factors: something you know (password), have (token/phone), are (biometric), and somewhere you are (location).
  • Implementations: biometrics, hard/soft tokens, security keys (FIDO2).

Passwords & PAM

  • Password best practices — length (most important), complexity, no reuse, expiration, age.
  • Password managers, passwordless (passkeys, biometrics).
  • Privileged Access Management (PAM):
    • Just-in-time (JIT) permissions — grant elevated access only when needed, briefly.
    • Password vaulting — secure storage/checkout of privileged credentials.
    • Ephemeral credentials — short-lived, auto-expiring credentials.

4.7 Automation & Orchestration (Secure Operations)

Use Cases

User/resource provisioning, guard rails, security groups, ticket creation, escalation, enabling/disabling services, continuous integration & testing (CI/CD), integrations & APIs.

Benefits

  • Efficiency / time saving, enforcing baselines, standard infrastructure configurations, secure scaling, employee retention, faster reaction time, workforce multiplier.

Other Considerations

  • Complexity, cost, single point of failure, technical debt, ongoing supportability.

4.8 Incident Response

IR Process (memorize the order!)

  1. Preparation — policies, tools, training before anything happens.
  2. Detection — identify a potential incident.
  3. Analysis — confirm and scope it.
  4. Containment — limit the spread/damage.
  5. Eradication — remove the threat (malware, accounts).
  6. Recovery — restore systems to normal.
  7. Lessons Learned — post-incident review to improve.

Supporting Activities

  • Training and testing — tabletop exercises, simulations.
  • Root cause analysis (RCA) — find the underlying cause.
  • Threat hunting — proactively search for hidden threats.

Digital Forensics

  • Legal hold — preserve data relevant to litigation.
  • Chain of custody — documented handling of evidence (who, what, when).
  • Acquisition — collect evidence (image disks/memory) — capture volatile data first (order of volatility).
  • Reporting, preservation, e-discovery.

4.9 Data Sources to Support an Investigation

Log Data

  • Firewall logs — allowed/blocked connections.
  • Application logs — app-specific events.
  • Endpoint logs — host-level activity.
  • OS-specific security logs — Windows Security log, Linux auth logs.
  • IDS/IPS logs — detected/blocked intrusions.
  • Network logs — switch/router/flow data.
  • Metadata — data about data (email headers, file timestamps).

Other Data Sources

  • Vulnerability scans, automated reports, dashboards, packet captures (Wireshark/tcpdump — full content).
