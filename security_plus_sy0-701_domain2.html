2.1 Threat Actors and Motivations

Threat Actor Types

ActorDescriptionSophisticationResources
Nation-StateGovernment-sponsored APTs (Advanced Persistent Threats) — espionage, sabotageVery HighVery High
Unskilled Attacker (Script Kiddie)Uses pre-made tools/exploits without deep knowledgeLowLow
HacktivistPolitically/socially motivated (e.g., Anonymous)MediumLow–Medium
Insider ThreatCurrent/former employee, contractor, or partner with accessVariesInternal access
Organized CrimeProfit-driven; ransomware gangs, fraud ringsHighHigh
Shadow ITInternal users deploying unauthorized tech (not malicious, but risky)Low–MediumInternal

Actor Attributes

  • Internal vs. external — insider threats already have access; external must penetrate the perimeter.
  • Resources / funding — nation-states & organized crime are well-funded.
  • Level of sophistication / capability — APT > organized crime > hacktivist > script kiddie.

Motivations

  • Data exfiltration — stealing data (IP, PII, secrets).
  • Espionage — intelligence gathering (corporate or state).
  • Service disruption — taking down systems (often DDoS).
  • Blackmail — extortion using stolen data.
  • Financial gain — ransomware, fraud, theft.
  • Philosophical / political beliefs — hacktivism.
  • Ethical — authorized researchers/penetration testers.
  • Revenge — disgruntled insiders.
  • Disruption / chaos — pure destruction (some nation-states, some script kiddies).
  • War — cyberwarfare during armed conflict.

2.2 Threat Vectors and Attack Surfaces

Message-Based Vectors

  • Email — phishing, malicious attachments, links.
  • SMS / IM — smishing or malicious links via text/instant messaging.

Other Delivery Vectors

  • Image-based — malicious payload hidden in images (e.g., steganography, exploit in image parser).
  • File-based — malicious documents, executables, scripts.
  • Voice call — vishing.
  • Removable device — USB drops, malicious flash drives.

Software / System Vectors

  • Vulnerable software — unpatched apps. Client-based needs an installed agent; agentless uses standard protocols (e.g., SSH, WinRM).
  • Unsupported systems — past end-of-life; no patches.
  • Unsecure networks — open Wi-Fi, weak WPA, exposed wired ports, vulnerable Bluetooth (e.g., BlueBorne).
  • Open service ports — extra attack surface (Telnet, SMB, RDP).
  • Default credentials — unchanged factory passwords on routers, IoT, databases.

Supply Chain Vectors

  • Managed Service Providers (MSPs) — compromise of MSP can fan out to many clients.
  • Vendors / Suppliers — tampered hardware or malicious software updates (e.g., SolarWinds-style).

Human Vectors / Social Engineering

  • Phishing — fraudulent email seeking credentials/clicks.
  • Vishing — voice-call phishing.
  • Smishing — SMS phishing.
  • Spear phishing — targeted phishing; whaling — targets executives.
  • Misinformation / disinformation — false content used to deceive or influence.
  • Impersonation — pretending to be someone else (helpdesk, exec).
  • Business Email Compromise (BEC) — attacker uses (or spoofs) a legitimate exec's mailbox to authorize wire transfers/data.
  • Pretexting — fabricated scenario/story to extract info.
  • Watering hole — compromise a site frequented by the target group.
  • Brand impersonation — fake site/email mimicking a trusted brand.
  • Typosquatting — registering misspelled domains (e.g., gooogle.com).

2.3 Types of Vulnerabilities

Application Vulnerabilities

  • Memory injection — injecting code into a running process's memory.
  • Buffer overflow — writing past allocated memory; can overwrite return addresses to execute attacker code.
  • Race conditions — timing flaws between Time-of-Check (TOC) and Time-of-Use (TOU).
  • Malicious update — compromised software update channel delivers malware.

Operating System (OS)-Based

  • Unpatched kernel/system services, escalation flaws, vulnerable drivers.

Web-Based

  • SQL Injection (SQLi) — attacker inserts SQL into input to manipulate DB queries.
  • Cross-Site Scripting (XSS) — attacker injects client-side scripts that run in another user's browser. (Reflected, Stored, DOM-based.)

Hardware

  • Firmware — vulnerable BIOS/UEFI/device firmware.
  • End-of-life (EOL) — manufacturer no longer issues fixes.
  • Legacy — old hardware/software still in production with known weaknesses.

Virtualization

  • VM Escape — breaking out of a guest VM into the hypervisor or other VMs.
  • Resource Reuse — leftover data in memory/storage exposed to a new tenant.

Cloud-Specific

  • Misconfigured storage buckets, weak IAM, exposed APIs, shared-tenancy risks.

Supply Chain

  • Service provider — MSP/SaaS compromise.
  • Hardware provider — counterfeit or implanted components.
  • Software provider — trojaned updates or libraries.

Cryptographic Vulnerabilities

  • Weak algorithms (MD5, SHA-1, DES), short keys, poor RNG, improper implementation, expired certs.

Misconfiguration

  • Default settings, open S3 buckets, weak permissions, debug endpoints, verbose error messages.

Mobile Device

  • Sideloading — installing apps outside official app stores.
  • Jailbreaking / Rooting — removing OS restrictions, bypassing security model.

Zero-Day

A vulnerability unknown to the vendor — no patch exists. Highly valuable to attackers.

2.4 Indicators of Malicious Activity

Malware Attacks

TypeWhat It Does
RansomwareEncrypts files / locks systems and demands payment.
TrojanDisguised as legitimate software; opens backdoors.
WormSelf-replicating; spreads across networks without user action.
SpywareSecretly collects data on user activity.
BloatwarePre-installed unneeded software; performance/security risk.
VirusAttaches to a host file; needs execution to spread.
KeyloggerRecords keystrokes (passwords, messages).
Logic BombTriggers malicious action on a condition (date, event).
RootkitHides at OS/kernel level, evading detection; persistence + privilege.

Physical Attacks

  • Brute force — physically forcing entry (also: brute-forcing a lock).
  • RFID cloning — duplicating an RFID badge to gain access.
  • Environmental — tampering with HVAC, power, fire suppression to disrupt ops.

Network Attacks

  • DDoS — distributed denial of service; amplified uses small request → large response from open services (NTP, DNS); reflected spoofs source IP so responses go to the victim.
  • DNS attacks — DNS poisoning, cache poisoning, hijacking, domain takeover.
  • Wireless — rogue AP, evil twin, deauth, WPS attacks.
  • On-path (formerly MitM) — attacker intercepts/modifies traffic between two parties.
  • Credential replay — captured credentials/sessions resent to authenticate.
  • Malicious code — payload delivered over the network (drive-by, exploit).

Application Attacks

  • Injection — SQLi, command injection, LDAP injection.
  • Buffer overflow — overwrites memory.
  • Replay — capturing a request/token and resending it.
  • Privilege escalation — vertical (user→admin) or horizontal (user→another user).
  • Forgery — CSRF, request forgery, token forgery.
  • Directory traversal — using ../ to access files outside the web root.

Cryptographic Attacks

  • Downgrade — forcing weaker protocol (e.g., TLS 1.0 instead of 1.3).
  • Collision — finding two inputs that hash to the same value.
  • Birthday — probability-based collision attack (relies on the birthday paradox).

Password Attacks

  • Spraying — trying ONE common password against MANY accounts (avoids lockout).
  • Brute force — trying every possible combination on ONE account.

Indicators of Compromise (IoCs)

  • Account lockout — repeated lockouts may indicate brute force / spraying.
  • Concurrent session usage — same user logged in from two places at once.
  • Blocked content — security tools blocking unusual URLs/files.
  • Impossible travel — login from NY, then Tokyo 10 minutes later.
  • Resource consumption — unusual CPU/RAM/network spikes.
  • Resource inaccessibility — files encrypted (ransomware) or services down.
  • Out-of-cycle logging — log entries at unusual times.
  • Published / documented — the threat appears on threat intel feeds / CVEs.
  • Missing logs — attacker cleared logs to hide tracks.

2.5 Mitigation Techniques to Secure the Enterprise

Core Mitigations

  • Segmentation — divide the network (VLANs, subnets, microsegmentation) to limit lateral movement.
  • Access control — ACLs on routers/firewalls/files, fine-grained permissions on resources.
  • Application allow list — only approved binaries can run; blocks unknown malware.
  • Isolation — separating systems entirely (air-gapped, sandbox).
  • Patching — timely application of vendor updates.
  • Encryption — protect data at rest and in transit.
  • Monitoring — SIEM, EDR, IDS/IPS, log analysis.
  • Least privilege — give users / services only the access they need.
  • Configuration enforcement — automated tools (GPO, Ansible, Intune) ensure baselines stay in place.
  • Decommissioning — securely retire old systems; wipe storage, revoke certs/accounts.

Hardening Techniques

  • Encryption (full-disk, BitLocker/FileVault).
  • Endpoint protection — antivirus / EDR.
  • Host-based firewall — controls per-host network traffic.
  • HIPS (Host-based Intrusion Prevention System) — blocks suspicious behavior on the host.
  • Disable unused ports / protocols — close attack surface (Telnet, SMBv1).
  • Default password changes — replace factory creds on every device.
  • Removal of unnecessary software — fewer apps = fewer vulnerabilities.
